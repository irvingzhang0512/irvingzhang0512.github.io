📡 🔥 Skill/MCP 腰部热门（5 条）

ZaxbyHub/opencode-swarm

面向架构师的 OpenCode 多智能体协作插件，通过中心化调度+专家咨询+代码生成+QA 审查的 hub-and-spoke 架构，实现高质量、可控的 AI 辅助软件开发。

✍️ ZaxbyHub

要点

  • Hub-and-Spoke 多智能体编排：采用中心调度节点协调多个专业子智能体（需求分析 SME、代码生成器、QA 审查器），相比单一 Agent 能更好地处理复杂软件工程任务的分解与质量把控

wlsdks/promptlane

PromptLane 是一个本地优先的提示词改进工作区，专为 Codex、Claude Code 等长时间运行的编码 Agent 设计，通过结构化流程和 SQLite 本地存储帮助开发者系统化地优化提示词效果。

✍️ wlsdks

要点

  • 本地优先与隐私保护：采用 SQLite 本地存储所有提示词和迭代记录，无需依赖云端服务，确保敏感代码和提示词数据完全由开发者掌控

ebongard/renfield

Renfield 是一个可完全离线运行的自托管 AI 助手，集成语音卫星设备、100+ 工具和 RAG 知识库，专注于隐私保护和本地化智能家居控制。

✍️ ebongard

要点

  • 离线优先与隐私保护：基于 Ollama 实现完全本地运行，支持语音识别、说话人识别和存在检测，所有数据处理均在本地完成

brilliantdirectories/brilliant-directories-mcp

这是 Brilliant Directories 官方推出的 MCP 服务器，让 AI Agent 能直接管理会员、帖子、线索、评论等网站运营内容，将 AI 助手变为自动化网站管理员。

✍️ brilliantdirectories

要点

  • 全功能网站管理接口：通过 MCP 协议暴露 Brilliant Directories 平台的核心功能（会员管理、内容发布、线索处理、评论审核等），让 Claude、Cursor 等 AI Agent 直接操作网站后台

alextra-lab/personal_agent

这是一个探索认知架构的本地AI协作者研究项目，融合了图谱记忆、工具使用与多模型协调，值得关注其如何将LLM思维过程结构化。

✍️ alextra-lab

要点

  • 构建了基于Neo4j的“认知架构”，将LLM的思维过程建模为知识图谱，增强推理可解释性与长期记忆

📡 🎯 Skill/MCP 每日精选（5 条）

metorial/metorial-platform

这是一个面向 MCP（Model Context Protocol）生态的连接管理平台，为数十万 MCP 连接提供引擎支持，值得关注因其瞄准了 Agent 工具规模化部署与运维的痛点。

✍️ metorial

要点

  • MCP 连接即服务：将 MCP Server 的连接管理、托管和分发作为 SaaS 平台提供，降低了开发者自建 MCP 基础设施的复杂度

fdueblab/Micro-Agent

这是一个专为垂直领域（如医疗、法律、金融）设计的轻量级 AI Agent 框架，通过“微型代理”模式将复杂任务拆解为可复用的原子化工具，适合需要快速落地行业场景的开发者。

✍️ fdueblab

要点

  • 技术亮点：基于 FastAPI + LiteLLM 构建，原生支持 MCP 协议和 RAG 能力，可灵活接入多种大模型；采用 ReAct 模式实现工具调用与任务规划，代码结构清晰、易于扩展。

TNT-Likely/BeeCount

BeeCount 是一个隐私优先的本地记账应用，通过集成 MCP 协议实现 AI 自动记账，是个人财务工具与 Agent 能力结合的优秀范例。

✍️ TNT-Likely

要点

  • MCP 驱动的 AI 记账：支持 Model Context Protocol，允许 AI 助手（如 Claude Desktop）直接接入记账数据，实现语音/文本驱动的自动记账和查询。

ucsandman/DashClaw

DashClaw 是 AI Agent 的治理运行时，通过拦截行为、执行防护策略、要求审批并生成审计就绪的决策轨迹，为 Agent 操作提供企业级安全与合规保障。

✍️ ucsandman

要点

  • Agent 行为拦截与策略强制执行：运行时拦截 Agent 的每一次操作，根据可配置的防护策略（guard policies）进行实时决策，支持阻断、审批或放行，确保 Agent 行为符合组织规范。

oomol-lab/open-connector

Open Connector 是一个开源的身份认证网关，通过统一 OAuth 层让 AI Agent 能安全连接 1000+ SaaS 服务，解决了 Agent 工具调用中的授权碎片化难题。

✍️ oomol-lab

要点

  • 多协议接入：同时支持 SDK、CLI、MCP、HTTP 和 OpenAPI，Agent 可选择最自然的方式集成，尤其 MCP 原生支持使其可直接作为模型上下文协议工具使用。
