h1ddenpr0cess20/wordmark

一个纯客户端的 AI 聊天应用，支持 OpenAI、xAI 等云端 API 及本地模型，内置工具调用、MCP 和 TTS，无需后端，适合关注本地优先与多模型集成的开发者。

✍️ h1ddenpr0cess20

要点

  • 技术亮点：完全前端运行，无后端依赖，集成 MCP（Model Context Protocol）支持工具调用，同时兼容 OpenAI Responses API 和 xAI Grok，并包含本地 LM Studio/Ollama 模型接入。

Sanjit-K/ftc-toolchain

这是一个面向 FIRST Tech Challenge 机器人竞赛的 MCP 工具链，让 AI 助手能直接参与机器人的开发、部署和调试，值得关注因为它将 MCP 协议应用到了实体机器人开发领域。

✍️ Sanjit-K

要点

  • 技术亮点：通过 MCP 协议桥接 AI 助手（Claude、Codex）与 FTC 机器人开发流程，支持代码生成、部署和调试的自动化

Drykeeightyeight427/MinecraftOptimizer

这是一个通过系统级调整（非游戏内模组）一键优化 Minecraft Java 版帧率的工具，值得关注的是它将“游戏优化”包装成了类似 Agent 工具的可执行命令。

✍️ Drykeeightyeight427

要点

  • 技术亮点：采用 Gradle Kotlin DSL 构建，集成对 Fabric/Quilt/Sponge 等多种模组加载器以及 Docker 的支持，表明它可能通过容器化或构建脚本实现环境清理与性能调整。

ModernRelay/omnigraph

Omnigraph 是一个湖仓原生的图引擎，支持 Git 式版本控制工作流，为 AI Agent 提供了可查询、可回溯的知识图谱基础设施。

✍️ ModernRelay

要点

  • 湖仓原生与版本控制：基于 DataFusion 和 Lance 构建，将图数据直接存储在 S3 等对象存储上，并内置类似 Git 的分支、提交、回滚机制，让 Agent 对知识图谱的操作可追溯、可协作。

AgentVisa-ai/mcp

AgentVisa 是一个 MCP 服务器与 Claude Code 插件，让 AI Agent 通过出示已验证的人类身份令牌来访问受保护网站，解决 Agent 被网站屏蔽的身份验证问题。

✍️ AgentVisa-ai

要点

  • 技术亮点：将“已验证人类身份”封装为可出示的令牌，通过 MCP 协议与 Claude Code 集成，为 AI Agent 提供了一种标准化的身份证明机制，使其能通过网站的反爬/反机器人检测

📡 🎯 Skill/MCP 每日精选（5 条）

hi-godot/godot-ai

这是一个为 Godot 游戏引擎提供的生产级 MCP 服务器和 AI 工具插件，让 AI 助手能直接操控和创建游戏内容，值得关注因为它打通了 AI 与游戏引擎的实时交互通道。

✍️ hi-godot

要点

  • 技术亮点：以 MCP 服务器形式运行，支持一键安装，提供 20+ 工具让 AI 直接操作 Godot 编辑器（创建场景、添加节点、编辑脚本、控制运行等），实现自然语言驱动的游戏开发。

ZaxbyHub/opencode-swarm

面向 OpenCode 的架构师中心化多智能体编码插件，通过“中心-轮辐”模式协调领域专家、代码生成和 QA 审查，值得关注其将架构决策权交还人类的协作式 AI 编码新思路。

✍️ ZaxbyHub

要点

  • 架构师中心的“中心-轮辐”编排：人类架构师通过 OpenCode CLI 下达高层指令，Swarm 自动分解任务、调用专业子代理（SME 咨询、代码生成、代码审查）并行工作，最后汇总结果，确保架构意图不被 AI 偏离。

Bande-a-Bonnot/Boucle-framework

Boucle 是一个用 Rust 编写的自主 Agent 框架，专注于结构化记忆、安全钩子和循环管理，且由运行在它之上的 Agent 自身构建，体现了“自举”式开发理念。

✍️ Bande-a-Bonnot

要点

  • 结构化记忆与安全钩子：框架内置了结构化的记忆系统和可配置的安全钩子（safety hooks），用于控制 Agent 的行为边界，防止失控操作。

nextlevelbuilder/goclaw

GoClaw 是用 Go 重写的 AI Agent 编排框架，主打多租户隔离、五层安全架构和原生高并发，适合企业级大规模部署安全可控的智能体团队。

✍️ nextlevelbuilder

要点

  • 企业级安全与隔离：提供五层安全模型和多租户隔离，解决 AI Agent 在生产环境中权限混乱、数据泄漏的痛点，适合对安全有严格要求的组织。

luckyPipewrench/pipelock

Pipelock 是一个开源的 AI Agent 防火墙，专注于 MCP 协议安全与 Agent 出口流量监控，能扫描 HTTP/MCP/A2A/WebSocket 流量以防止数据泄露、SSRF 和提示注入攻击，并提供可验证的审计凭证。

✍️ luckyPipewrench

要点

  • 技术亮点：作为中介代理部署在 Agent 和外部服务之间，支持 MCP、A2A 等新兴 Agent 协议，能实时扫描外发流量并生成中介签名的”操作回执”，为 Agent 行为提供不可篡改的外部审计证据。
