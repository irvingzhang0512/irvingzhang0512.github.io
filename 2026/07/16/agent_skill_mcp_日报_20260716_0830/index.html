📡 🔥 Skill/MCP 腰部热门（5 条）

xxpixel/claude-context-manager

这是一个为 AI 编程会话提供上下文管理能力的工具，能够保存、恢复和搜索编码上下文，解决 AI 辅助开发中上下文丢失和重复解释的痛点。

✍️ xxpixel

要点

  • 核心能力：支持保存和恢复 Claude 等 AI 编程助手的会话上下文，让开发者可以跨会话保持项目记忆，避免每次对话都从零开始解释项目结构。

sleep2agi/agent-network

一个本地优先的多 Agent 协作框架，支持一行命令将 Claude Code、Codex CLI 等主流 AI 编程工具组网为数字员工军团，并通过 Web Dashboard 统一监控管理。

✍️ sleep2agi

要点

  • 多运行时互通：支持 Claude Code、Claude Agent SDK、Codex CLI、Grok Build 四种运行时，打通 Anthropic、OpenAI、xAI、DeepSeek、GLM、Kimi 等 8+ 家 LLM，实现异构 Agent 在同一网络中协作。

dykeruv/argus-mcp

Argus MCP 是一个基于零信任模型的 AI 代码审查服务器，通过 MCP 协议为 Claude、Cursor 等 AI 编程工具提供安全、高效的代码质量审查能力。

✍️ dykeruv

要点

  • 零信任安全模型：在代码审查流程中引入零信任架构，确保 AI 对代码库的访问遵循最小权限原则，降低敏感代码泄露风险

AudiRamadyan/claude-swarm

这是一个用于编排多个 Claude Code 实例并行工作的 Swarm 框架，通过状态持久化和行为治理实现高效自主编程。

✍️ AudiRamadyan

要点

  • 多智能体并行编码：允许同时运行多个 Claude Code 工作进程，通过 Swarm 架构实现任务分解与并行处理，提升复杂编码任务的效率

HypaSMarty/SUMO-MCP-Server

这是一个将 SUMO 交通仿真器封装为 MCP Server 的项目，让 AI Agent 能够直接模拟和控制智能交通系统，值得关注其将专业仿真工具接入 Agent 生态的思路。

✍️ HypaSMarty

要点

  • 技术亮点：基于 SUMO（城市交通仿真标准工具）构建 MCP 接口，使 LLM 可通过标准化协议调用交通仿真能力，如信号灯控制、车流模拟等。

✍️ zoedsoupe

要点

这是一个社区驱动的“Claude 能力发现中心”，聚合了 Skills、MCP 服务器、Agents、Commands 等可复用组件，是 Claude 生态中类似“应用商店”的集散地。

✍️ davepoon

要点

  • 技术亮点：以 TypeScript CLI 工具形式提供统一入口，支持跨 Claude Code、Desktop、Agent SDK 和 OpenClaw 多平台的能力发现与安装，覆盖 Skills、Commands、Hooks、Plugins 等扩展类型。

av/harbor

Harbor 是一个一站式本地 AI 环境管理器，通过单一命令行工具将 LLM、向量数据库、MCP 服务等数百个组件预先编排好，让开发者无需手动配置即可快速启动完整的 AI 开发栈。

✍️ av

要点

  • 开箱即用的 AI 全栈环境：通过 Docker Compose 预配置了 LLM 后端、MCP 服务器、Web UI、向量数据库等数百个服务，一条 harbor up 命令即可在本地运行完整 AI 工作流

silverstein/minutes

这是一个开源的、隐私优先的对话记忆层，能够将会议、想法和语音笔记转录并存储，使其可被 AI 搜索和检索，为个人和团队构建第二大脑提供基础设施。

✍️ silverstein

要点
